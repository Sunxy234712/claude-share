# CLAUDE.md

本文件为 Claude Code（claude.ai/code）在此代码库中工作时提供指导。

## 重要文件

- `prompt.md`：各功能模块的开发提示词记录，按模块编号排列（00 登录注册 → 22 系统参数维护），每完成一个模块即补充对应提示词。

## 技术栈

| 层次 | 技术 |
|------|------|
| 前端 | Vue 3 + Element Plus |
| 后端 | Node.js + Express（或 Fastify） |
| 数据库 | Neon（Serverless PostgreSQL，免安装云端托管） |
| ORM | Drizzle ORM 或 Prisma（适配 Neon） |

### 目录结构约定

```
/
├── frontend/          # Vue 3 前端项目（Vite 构建）
│   ├── src/
│   │   ├── views/     # 页面组件，按业务模块分目录
│   │   ├── components/# 公共组件
│   │   ├── api/       # 后端接口封装（axios）
│   │   ├── store/     # Pinia 状态管理
│   │   └── router/    # Vue Router 路由
├── backend/           # Node.js 后端项目
│   ├── routes/        # 路由，按业务模块拆分
│   ├── controllers/   # 业务逻辑
│   ├── db/            # 数据库连接与 schema
│   └── middleware/    # 鉴权、日志、授权复核等中间件
└── CLAUDE.md
```

### 常用命令

```bash
# 前端
cd frontend
npm install
npm run dev          # 开发服务器
npm run build        # 生产构建

# 后端
cd backend
npm install
npm run dev          # 开发模式（nodemon）
npm run start        # 生产启动

# 数据库（Neon + Drizzle）
npx drizzle-kit generate   # 生成迁移文件
npx drizzle-kit migrate    # 执行迁移
npx drizzle-kit studio     # 可视化数据库管理界面
```

## 项目概述

本项目是**银行柜面系统**，即银行网点办理线下业务的核心平台。系统覆盖柜面全量业务，包括个人业务、对公业务、贷款授信、支付结算、特殊业务、内控管理和公共辅助七大功能域。

## 业务架构

### 一、个人业务
- **账户管理**：开户、销户、挂失/解挂、补卡/换卡、密码修改、客户信息维护、手机号变更
- **存取款与转账**：现金存取款、大额存取款登记、行内/跨行转账、汇款、定期/活期互转
- **账户查询**：余额查询、明细查询、交易流水打印
- **银行卡业务**：密码重置、卡片激活、限额调整
- **理财与贵金属**：理财购买/赎回、基金申购赎回、国债买卖、贵金属积存/兑换/实物提取

### 二、对公业务
- **单位账户管理**：对公账户开户、销户、变更、年检、印鉴管理、法人/经办人信息变更
- **对公结算**：支票（现金/转账）、电汇、银行汇票、本票、承兑汇票
- **代发代扣**：代发工资、批量转账、批量代扣费用
- **单位存款**：单位定期、通知存款、协定存款

### 三、贷款与授信
- 个贷/对公贷款发放、还款、展期、结清
- 抵质押物登记与解押
- 贷款查询、合同管理

### 四、支付结算与票据
- 大小额支付系统业务（CNAPS）
- 同城票据交换、提入/提出处理
- 票据挂失、止付、贴现

### 五、特殊业务
- **反洗钱与风控**：大额/可疑交易上报、身份识别
- **外汇业务**：结售汇、境外汇款、外币存取
- **社保卡/医保卡**：发卡、激活、信息变更
- **第三方存管**：证券资金账户签约、转账

### 六、内控与管理
- 柜员签到/签退、钱箱管理、尾箱现金调拨
- 柜员权限管理、操作日志审计
- 业务授权（大额、风险业务复核）
- 日终轧账、对账、清算、结息

### 七、公共辅助
- 凭证打印、回单打印、存折补登
- 客户排队叫号联动、影像采集（身份证/票据拍照）
- 系统参数维护、利率汇率维护
- 差错处理、冲正、撤销交易

## 核心领域术语

| 术语 | 说明 |
|------|------|
| 柜员 | 银行柜台操作员，具有权限等级，每日须签到/签退 |
| 授权/复核 | 高风险或大额交易需主管审批，实行双人操作 |
| 尾箱 | 柜员日常现金抽屉，日终须轧平 |
| 轧账 | 日终对所有交易进行对账平衡的操作 |
| 大小额支付 | 中国现代化支付系统（CNAPS），用于跨行转账 |
| 印鉴 | 对公账户身份验证所用的签名/印章 |
| 冲正/撤销 | 因差错对交易进行反向冲销或取消 |
| 大额现金 | 超过监管阈值须登记上报的大额现金交易 |

## 重要业务规则

- 大额现金交易须登记，并可能触发反洗钱上报
- 高风险操作须双人授权（经办+复核）
- 所有柜员操作须记录时间戳和操作员 ID，用于审计
- 日终尾箱差额为零方可完成签退
- 开户及特定交易须强制核验客户身份
